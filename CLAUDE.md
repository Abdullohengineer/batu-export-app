# Working on this repo

## Git workflow

- **Never commit directly to `main`.** Always work on a feature branch off `main` — one branch per phase or feature (e.g. `phase1-manager-kirim`, `fix-gate-weighing-bug`).
- Push the branch and let the user review and merge via PR. Don't merge to `main` yourself unless explicitly asked.
- If `main` doesn't exist yet or you're unsure what it points to, ask before creating branches or pushing, rather than guessing.
